项目布局

当程序慢慢变大 将所有的路由，视图函数 都放在一个文件，共享app时，维护起来就不怎么舒服了，后续增加，阅读代码也很费劲，因此需要进行拆分。

当我们的代码有 user模块，admin模块以及各种各样的模块时，都写在一个文件main.py里面时，就会使得后期维护成本加大很多，不利于后期代码的增删改查：

循环引用

因此需要进行拆分，一开始拆分我是直接把模块新建python文件user.py,admin.py,代码分别是。

admin.py

user.py

此时 main模块的代码就变少了
main.py

但是 当我们启动时就报错了！！！

我们分析下，

首先运行main.py，会先去读取main.py的内容，从上往下开始读，当读取到from admin import admin_login,admin_logout 这句话时，会去admin模块查找,从上往下查找，在admin模块中又存在from main import app这一句话,因此程序又会去读取main模块，继续从上往下读，然后又读到了from admin import admin_login,admin_logout，又回到了admin模块，不断的在两个模块之间循环的读取引用，谁也让谁，导致程序执行出错。
